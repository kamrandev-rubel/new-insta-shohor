Javascript is a scripting language for the web. It is an interpreted language which means it does not need to compile or translate the code. Javascript does not work without a browser. The web browser loads a web page, parses the HTML, and creates a Document Object Model for the content. The HTML and CSS are put together by the DOM to create the web page first. Then, the browsers' JavaScript engine loads JavaScript files and inline code but does not run the code immediately. It waits for the HTML and CSS to finish loading. Once this is done, the JavaScript is executed in the order the code is written. This results in the DOM being updated by JavaScript code and rendered by the browser.

Local Storage: It keeps storing the user information data without expiration date this data will not be deleted when a user closes the browser windows. It will be available for days, weeks, months, and years. Local storage has 4 methods: 1. setItem Method 2. getItem Method 3. removeItem Method 4. Clear Method
Session Storage: Session Storage objects can be accessed using the sessionStorage read-only property. The sessionStorage data is cleared when the page session ends. Session Storage property maintains a separate storage area for each given origin that's available for the duration of the page session i.e as long as the browser is open, including page reloads and restores. Session Storage has 4 methods: 5. setItem Method 6. getItem Method 7. removeItem Method 8. Clear Method

The event loop has one main function, at every tick of the javascript run process, it looks at the callstack and checks if any function or activity is available, if none is available it pushes items from the queue into it, then that operation is later run on the stack.
